Float Slide Editor

HTML 슬라이드를 편집 가능한 Flat 요소로 변환하는 브라우저 기반 슬라이드 편집기

React 19 + Vite 8 · Python (FastAPI + python-pptx) · 21,000+ lines
1

왜 만들었는가?

기존 도구의 한계와 Float Editor의 접근

기존 문제

  • Claude/GPT가 생성한 HTML 슬라이드를 편집할 수단이 없음
  • HTML → PPT 변환 시 레이아웃/폰트가 깨짐
  • 브라우저에서 WYSIWYG 편집 후 다양한 포맷으로 내보내기 불가

Float Editor 해결

  • HTML을 Flat 요소로 자동 변환 (절대 좌표 + computed style)
  • 변환 후 PowerPoint처럼 직접 편집
  • PPT/HTML/PNG/JSON 다중 포맷 내보내기
  • 폰트 임베딩으로 PPT 95%+ 픽셀 일치

FlatExtractor — HTML→Flat 변환

iframe DOM을 순회하여 시각적 말단 요소를 FlatElement[]로 추출

주요 처리

  • 슬라이드 프레임워크 감지 (reveal.js, scroll-snap, opacity 기반)
  • Flex 부모 → 자식 독립 추출 (텍스트 노드 Range API)
  • 배경 체인 탐색 → 다중 레이어 배경 생성
  • 전체 크기 shape/image → 배경으로 자동 병합
  • 코드 블록 자동 감지 (isCode: true)
  • @font-face / Google Fonts URL 수집

품질 규칙

  • 텍스트 중복 방지 (부모/자식 dedup)
  • 네비게이션 요소 자동 제외
  • nowrap 보정 (줄바꿈 방지)
  • overflow:hidden 클리핑 + border-radius 상속
  • font-variation-settings wght 축 해석
  • CSS 클래스 스타일 비교 (span 고유 스타일 보존)

스냅 가이드 시스템

SnapEngine.js — 순수 함수 기반 스냅 계산

가이드 유형

정렬 ━━━ 빨강 실선 — 요소 가장자리/중심 정렬
간격 ←12px→ 분홍 선 + px 라벨 — 인접 요소와의 거리
균등 - - - 보라 점선 — 요소 간 동일 간격 스냅
크기 - - - 파랑 점선 — 인접 요소와 동일 크기 스냅

구현 특징

  • 순수 함수 — 사이드이펙트 없음, 테스트 용이
  • 이동/리사이즈 모두 지원
  • 캔버스 경계 + 다른 요소 참조점
  • 단일/그룹 선택 모두 적용
  • threshold 5px (설정 가능)

Flat 변환 품질 처리

텍스트 정확도

  • Flex 부모 텍스트 노드 → Range API로 독립 추출
  • CSS 클래스 스타일 비교: <span> 고유 스타일 보존
  • font-variation-settings wght 축 → fontWeight 해석
  • font-feature-settings 따옴표 escape 처리
  • nowrap 보정: 줄바꿈 방지 너비 측정 + 부모 제약 감지
  • overflow:hidden 조상만 클리핑 (shrink-wrapped flex 제외)

레이아웃 정확도

  • 슬라이드 프레임워크 자동 감지 (reveal.js / opacity / scroll-snap)
  • 비활성 슬라이드 필터링 (isHiddenByAncestor)
  • 배경 체인 탐색: body → container → slide 순서
  • 전체 크기 shape/image → 배경 레이어 자동 병합
  • border-radius 상속 (overflow:hidden 부모에서)
  • 코드 블록 자동 감지: <pre>/<code> → isCode:true

폰트 복원

추출 단계

  • computedStyle.fontFamily에서 원본 복원
  • 인라인 style → CSSOM 규칙 → Google Fonts URL 순으로 탐색

CSS 수집

  • CSSOM 기반 @font-face / @import 추출
  • HTML 소스에서 googleapis URL 복구
  • 상대 URL → 절대 URL 변환

Flat 렌더링

  • fontImports를 부모 <head>에 주입
  • 동일 폰트로 캔버스 렌더링

PPT Export 품질 처리

발견된 문제 → 해결

  • 테마 그림자 → add_shape()의 <p:style> 제거
    effectRef idx=2가 테마 shadow 자동 적용
  • 유리 효과 → 미세 그래디언트 필터 강화 (alpha<0.3 스킵)
  • 그림자 과도 적용 → effective(alpha×blur) < 1.0 필터링
  • 배경 선택됨 → noSelect/noMove/noResize 잠금
    이후 마스터 슬라이드 배경으로 전환
  • 부분 border 누락 → 색상/굵기 다른 면은 개별 connector
  • 타원 오판 → aspect ratio 1.5:1 체크 추가

마스터 배경 전략

  • 모든 페이지 배경 fingerprint 수집
  • 가장 흔한 패턴 → 슬라이드 마스터 배경
  • 다른 페이지 → 슬라이드 레벨 override
  • 장식 레이어 (격자 등) → 잠금 shape 유지

폰트 임베딩 과정

  • Google Fonts CSS → 서브셋 감지 (124블록/weight)
  • GitHub에서 전체 variable font 다운로드
  • mutator.instantiateVariableFont + 테이블 제거
  • 내부 이름 수정 (Thin → Regular/Bold)
  • 사용 글자만 subsetting → 10MB → 80KB

반복 개선 사례

사용자 피드백 → 원인 분석 → 수정 → 검증 사이클

1

"텍스트가 다음 줄로 넘어간다"

width 버퍼(3%→8%) → 다른 레이아웃 파괴 → normAutofit 도입 (OOXML 자동 축소) + 버퍼 완전 제거

2

"보존 건드리지 않음" 텍스트 중복 추출

flex 부모 전체 텍스트 + 자식 span 각각 추출 → Range API로 텍스트 노드 독립 추출 + mergedContainerIds dedup

3

"1P 선에 그림자가 있다"

python-pptx add_shape()/add_connector()가 <p:style effectRef idx=2> 자동 삽입 → _clear_theme_style()로 생성 직후 제거

4

"폰트 이름이 Noto Sans KR Thin으로 표시"

Google Fonts 서브셋(34KB) 사용 + variable font default=100 → GitHub 전체 다운로드 + weight 인스턴스화 + 이름 테이블 수정

5

"배경이 검은색으로 보인다"

body 배경만 추출 + 슬라이드 자체 배경 누락 → 배경 체인 탐색 (revealPresent → 부모 순회) + 다중 레이어 추출
